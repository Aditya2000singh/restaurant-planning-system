# Restaurant Resource Planning System — Self-Learning Forecaster

A system that predicts how many customers a restaurant will get on any given day,
how many staff to schedule, and how much of each ingredient to order.

The key feature: when the manager says "we predicted 120 but only got 85 today",
the system learns from that correction and gets more accurate over time.

---

## What it does

**Prediction** — given tomorrow's day of week, weather and event info, it predicts:
- How many covers (customers) to expect
- Kitchen / floor / bar staff headcount
- Chicken, veggies and flour order quantities (with shelf life factored in)

**Self-learning** — every time a manager submits the real number of covers,
the model does a `partial_fit()` on that data point. No full retrain needed.
Over time the error shrinks — you can see this on the dashboard chart.

## How to run it

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Generate the dataset and train the model

```bash
cd src
python data_gen.py
python predictor.py
```

This creates `data/historical_data.csv` and saves the model to `models/`.

### 3. Run the dashboard (easiest way to use it)

```bash
streamlit run src/dashboard.py
```

Open your browser at `http://localhost:8501`

### 4. Or run the API separately

```bash
cd src
uvicorn api:app --reload
```

API docs at `http://localhost:8000/docs`

## API endpoints

| Method | Endpoint    | What it does                              |
|--------|-------------|-------------------------------------------|
| POST   | `/predict`  | Returns covers, staff, ingredients        |
| POST   | `/correct`  | Submits real covers — updates the model   |
| GET    | `/accuracy` | Returns full correction log with errors   |

### Example: predict

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"day_of_week": 5, "is_raining": false, "is_event": true}'
```

### Example: manager correction

```bash
curl -X POST http://localhost:8000/correct \
  -H "Content-Type: application/json" \
  -d '{
    "date": "2025-04-28",
    "day_of_week": 0,
    "is_raining": true,
    "predicted_covers": 120,
    "actual_covers": 85,
    "reason": "Heavy rain"
  }'
```

## How the self-learning works

1. Model trains on 180 days of historical data using `SGDRegressor`
2. Manager submits actual covers via the dashboard or API
3. `feedback.py` calls `model.partial_fit(X, [actual])` — updates weights in-place
4. Model is saved back to disk immediately
5. All corrections are logged to `data/corrections.csv`
6. The dashboard shows a chart of % error over time — it should trend downward

The model does not retrain from scratch every time. It just nudges its coefficients
toward the truth, which is more realistic for a live production system.

## Dataset

The dataset is generated by `src/data_gen.py`. It creates 180 rows covering
6 months of restaurant history with these columns:

- `date`, `day_of_week`, `is_raining`, `is_holiday`, `is_event`
- `actual_covers` — the target variable
- `kitchen_staff`, `floor_staff`, `bar_staff`
- `chicken_kg`, `veggies_kg`, `flour_kg`

Weekends have a higher base (140 covers vs 80 on weekdays).
Events add ~40, holidays add ~30, rain subtracts ~20, plus random noise.
